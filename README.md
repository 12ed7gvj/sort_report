# 前言  
本實驗透過 HTML5、CSS3 與 JavaScript 實作視覺化排序模擬器，探討五種經典排序演算法的運作機制。透過動態視覺呈現與時間統計，觀察不同演算法在處理相同數據規模時的表現差異。
# 氣泡排序法
原理： 重複走訪要排序的數列，一次比較兩個相鄰元素。  
操作： 若順序錯誤則交換，每輪會將當前最大值「浮」到數列末端。  
模擬實作內容:
用顏色區分相鄰元素、位置交換跟末端元素，有時間統計跟狀態控制的功能  
# 選擇排序法
原理： 在未排序區段中尋找最小值。  
操作： 找到後與未排序區的第一個元素交換，逐步縮小未排序範圍。  
模擬實作內容:
用顏色區分目標位置、比較的元素跟最小值，有時間統計跟狀態控制的功能  
# 插入排序法
原理： 類似整理撲克牌，將元素逐一插入已排序序列的正確位置。  
操作： 比對當前元素與左側已排序元素，若較小則將左側元素後移，最後置入空位。  
模擬實作內容:
用顏色區分子陣列區塊、資料寫回動作跟數值大小比較，有時間統計跟狀態控制的功能  
# 合併排序法
原理： 分治法 (Divide and Conquer)。將陣列對半切分至最小單位，再進行合併。  
操作： 核心在於 merge 過程，比較兩側子陣列的首項，依序填入暫存空間。  
模擬實作內容:
用顏色區分位置比對、插入的元素跟向後移動的元素，有時間統計跟狀態控制的功能  
# 快速排序法
原理： 選擇一個基準點 (Pivot)，將數列分為小於及大於基準點的兩部分。  
操作： 透過遞迴對分割出的子區塊重複執行，是實務上最常用的高效演算法。  
模擬實作內容:
用顏色區分基準值、移動的元素比較跟左右元素交換，有時間統計跟狀態控制的功能  
# 複雜度分析  
| 排序法 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 | 穩定性 | 實作視覺重點 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **氣泡排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 | 相鄰交換、末端固定 |
| **選擇排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 | 搜尋最小值、每輪一換 |
| **插入排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 | 元素平移、適合小量資料 |
| **合併排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 | 分治法、區域合併 |
| **快速排序** | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 不穩定 | 基準值 (Pivot) 分割 |

# 實驗設計  
技術棧： JavaScript 原生實作，利用 setTimeout 控制動畫頻率。  
計時機制： 程式於排序開始時記錄 startTime，完成後記錄 endTime，並計算差值顯示於網頁。  
資料生成： 每份檔案皆具備 reset() 函式，隨機生成 10-20 個元素的陣列作為基準。
