# 氣泡排序法
原理:重複比較相鄰元素並交換，每一輪將該輪最大的元素浮到頂部  
複雜度分析:  
時間複雜度: O(n²)  
空間複雜度: O(1)  
模擬實作內容:
用顏色區分相鄰元素、位置交換跟末端元素，有時間統計跟狀態控制的功能  
# 選擇排序法
原理:每一輪從未排序的部分中尋找最小值，並將其與未排序部分的第一個元素交換位置  
複雜度分析:  
時間複雜度: O(n²)  
空間複雜度: O(1)  
模擬實作內容:
用顏色區分目標位置、比較的元素跟最小值，有時間統計跟狀態控制的功能  
# 插入排序法
原理:將資料分為已排序與未排序兩部分，逐一將未排序的元素插入到已排序序列中的正確位置  
複雜度分析:  
時間複雜度: O(n²)  
空間複雜度: O(1)  
模擬實作內容:
用顏色區分位置比對、插入的元素跟向後移動的元素，有時間統計跟狀態控制的功能  
# 合併排序法
原理:採用分治策略，將數列遞迴拆解至最小單位，再將兩個已排序的子數列合併成一個更大的排序數列  
複雜度分析:  
時間複雜度: O(nlogn)  
空間複雜度: O(n)  
模擬實作內容:
用顏色區分子陣列區塊、資料寫回動作跟數值大小比較，有時間統計跟狀態控制的功能  
# 快速排序法
原理:選擇一個基準點，將數列分為小於基準與大於基準兩部分，再對這兩部分進行遞迴排序  
複雜度分析:  
時間複雜度: O(nlogn)  
空間複雜度: O(logn)  
模擬實作內容:
用顏色區分基準值、移動的元素比較跟左右元素交換，有時間統計跟狀態控制的功能  
# 綜合比較
| 排序法 | 平均時間複雜度 | 最差時間複雜度 | 空間複雜度 | 穩定性 | 實作視覺重點 |
| :--- | :---: | :---: | :---: | :---: | :--- |
| **氣泡排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 | 相鄰交換、末端固定 |
| **選擇排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 不穩定 | 搜尋最小值、每輪一換 |
| **插入排序** | $O(n^2)$ | $O(n^2)$ | $O(1)$ | 穩定 | 元素平移、適合小量資料 |
| **合併排序** | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | 穩定 | 分治法、區域合併 |
| **快速排序** | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | 不穩定 | 基準值 (Pivot) 分割 |
# 學習心得
這次的報告給了我很大的挑戰，但也給了我很大的收穫。無論是用github跟vsc做連結，還是用git做更進一步的程式，都讓我認為這次報告是個很好的學習。
我在這次報告學到了各個排序法的運作方式跟效率，一開始認為氣泡排序法最好用，但在數據變大後排序的時間比起合併排序法慢很多，我認識到在大數據時代下，選擇正確的演算法是性能的關鍵。我在報告中用了ai來製作各種不同的排序法，也讓它幫忙編排各個介面，讓排序法擁有很多種功能。
